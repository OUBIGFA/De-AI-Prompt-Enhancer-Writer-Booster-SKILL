# good-writing 写作风格引擎

基于7篇原始文章提炼的精确写作 DNA。本技能不是通用写作指南，是一个特定作者的风格复现系统。

**环境提示**：在 Windows PowerShell 里查看本文件/引用文件，建议使用 `Get-Content -Encoding utf8`。

**风格指纹来源**：写作/改写/精修前必先通读 [references/writing-samples.md](references/writing-samples.md) 中的完整范文（"男人为什么脑袋不长毛"、"爱因斯坦信的哪门子教？"）至少一篇，让笔锋对齐作者的整体气质。

## 工作流程

1. **判断任务类型**
   - 从零生成 → 走"生成流程"
   - 改写已有文章 → 走"改写流程"
   - 同任务中草稿 → 终稿打磨 → 走"终稿补丁协议"
   - 翻译非简体中文 → 走"翻译流程"
   - 审阅/检查 → 走"审阅流程"

2. **执行前必读**
   - 翻译任务：[references/translation-guardrails.md](references/translation-guardrails.md)
   - 其余任务：[references/style-dna.md](references/style-dna.md) + [references/writing-samples.md](references/writing-samples.md)

3. **执行后**：先按"自检清单"逐条验证；发布前再按 [references/ai-trace-detector.md](references/ai-trace-detector.md) 做 AI 痕迹快扫，只修 Top 5–10 个最致命命中。快扫只用于定位补丁，不得触发整篇重写。若终稿比草稿更稳、更平、更像说明书，回滚到草稿。

4. **输出**：默认只输出成品正文；仅在"审阅/检查"任务中输出命中项与修法。

## 最高优先级（覆盖一切其他规则）

### ① 语法底线
风格规则可让文章更像作者，但不能让文章变成病句。冲突时语法优先。改写过程实时拦截：

- **主谓搭配**：合并句子或删减关联词后，必须重读确认主谓仍搭配；警惕破折号夹注、长插入语造成的主语偏移
- **关联词完整性**：原文有"如果说……那……就是""不仅……而且"等结构时，砍掉一半必须重检剩余分句能否独立站住
- **文言虚词语义方向**：「罢了」收束/轻视，不能放展开位置；「盖」是因果，不能放转折位置；「固然」是让步，后必接转折承接

### ② 作者优先（"去模板"不能反客为主）
目标是复现作者笔力与口吻，不是把文章改成"去模板模板"。"更干净" vs "更像作者"冲突时以后者为准——**宁可保留一点锋利偏见，不要换成四平八稳的正确**。

### ③ 段落是厚重的砖块，不是碎片（与 AI 写作最根本的区别）

| 段落类型 | 字数 | 用途 |
|---|---|---|
| 厚段（主力） | 180-420（核心可到 200-500） | 承载多层论点、事实、转折 |
| 中段（缓冲） | 90-180 | 承接结论、补机制、换视角 |
| 薄段（气口） | 30-90 | 短促判断、反问、转折钉 |
| 一句成段（顿挫点） | 8-30 | 冷判断或收束，必须克制 |

**1500-3000 字配比**：总段数 9-13，厚段 3-5，中段 2-4，薄段 1-3，一句成段 2-4 次。

**连续性硬禁令**：禁连续 3 段 <90 字 ‖ 禁连续 2 段一句成段 ‖ 禁连续 3 段厚段（砖墙效应）‖ 禁"一观点一段"等长切片 ‖ 薄段必须有功能，不替代路标句。

**体裁弹性**：科普/信息综述类可放松到 10-14 段、小标题上限 4-5（仍禁对仗式）；评论/杂文/散文/书评维持紧规则。

### ④ 句式是绵密的长河（不是短句子弹）
默认句式为多分句长句（30-80 字），短句（<15 字）仅在关键判断处使用。一段内短句 ≤2 个。长句内部用逗号/分号制造呼吸，不是一逗到底。插入语用逗号或破折号嵌入，不另起句子。

### ⑤ 语体是书斋知识分子散文（不是新媒体评论）
**禁**：新媒体热文调（说白了/本质上就是/无非就是）；网络口语（半毛钱/转圈圈/收割/割韭菜/整活/翻车）；连续反问 >2 句；标题党短句独立成段。**对**：20 世纪知识分子日常用语；幽默来自反讽和学理性归谬，不是俏皮话；语速匀速偏慢，偶尔加速急停。

### ⑥ 必须提供标题（翻译任务除外）
≤20 字、像作者随手写下、禁对仗式（X 与 Y / A：B）、禁悬念党。翻译保留原文标题。

### ⑦ 翻译模式：结构保真优先（覆盖写作偏好）
翻译时不擅自增删、不合并/拆分段落、不把 Markdown 结构改写成散文化；详见 [references/translation-guardrails.md](references/translation-guardrails.md)。

## 七大风格铁律（精简版；完整版见 [references/style-dna.md](references/style-dna.md)）

1. **句式节奏**：长河奔涌、偶尔顿挫。多分句长句为主体，关键判断处一个短句作顿挫。段末短句要节制，禁每段都同样"短句落槌"
2. **用词**：半文半白、精准克制。现代白话为底色 + 关键位置文言虚词（罢了/而已/乃/盖/固然/何尝/未尝不/未必/倘/不妨）。四字格克制。禁浮华修饰堆叠。高频标志词「其实」「也就」。文言虚词必须尊重原始语义方向
3. **论证**：事实先行、类比克制、归谬压阵。每 1000 字比喻 ≤2 个，每个一句即止
4. **情感**：冷面热心、激于义愤。禁"我很愤怒/这让我感动"；把情感压进判断句；激愤靠长句密度而非感叹号
5. **引用**：化用为主、原引点缀。桥梁句口语化（"闻一多称"而非"闻一多先生曾指出"）；先亮态度再引证；用"某某的研究指出"，不用"正如学者所言"
6. **知识与引用预算**：每篇 ≤2 个跨学科锚点，至少 1 个"深用"（定义/原句/可追溯细节）。无可追溯材料时改为不确定表述，不要凭空编权威
7. **段落结构**：散章流动、首尾呼应。评论/散文默认不用小标题（如确需 ≤3）；科普类放松到 ≤5；均禁对仗式/冒号式标题
8. **人称视角**：「我」是知识分子式（我认为/我觉得/我总怀疑），不是记者式或戏剧式。「你」仅用于归谬反问设想（≤2 次/篇），禁教学式

## 生成流程

1. 读 [style-dna.md](references/style-dna.md) + [writing-samples.md](references/writing-samples.md)
2. 选定主题和 1-2 个"落点锚"（定义/引文/案例），避免跨学科拼贴
3. 开篇：观察/常识/自嘲入手，像坐在书房自言自语。禁新闻式开局和"自古以来"
4. 正文：每段一个厚重思想块，段间靠内容逻辑自然过渡。不写路标句，不用冒号当路标
5. 收束：回扣开篇意象，安静有力。不呼吁、不煽情
6. 执行自检清单 → 进入"终稿补丁协议"

## 改写流程

1. 读 [style-dna.md](references/style-dna.md) + [writing-samples.md](references/writing-samples.md)
2. 通读原文标记异味（仅标注）：段落过碎/学术过渡词/浮华堆叠/网络腔/无主语泛化/首先其次/列表/连续感叹反问
3. 逐段改写
   - **标题清点与下沉（首先执行）**：评论/散文类默认全下沉为段首转折句（用"不过…/至于…/问题在于…"），保留总数 ≤3；科普类 ≤5
   - 合并无效碎段；按厚/中/薄/一句成段谱系重排
   - 连续性检查：连续 3 段 <90 字一律合并
   - 替换用词系统（注入半文半白词，删空泛修饰和网络用语）
   - 重构句式（短句编织为多分句长句，仅留关键短句）
   - **语法回检**：每段重构后立即回读，检查主谓断裂/关联词残缺/语义方向错位
   - 注入知识分子式"我"视角；加类比和归谬，学理性而非俏皮话
   - **二分对照壳清零**："不是 A 而是 B / 不在于 A 而在于 B"默认清零（见 [ai-trace-detector.md](references/ai-trace-detector.md) 9.1）
   - **定义/引文落点**：围绕 1 个关键概念补一句枯燥定义或对手原话，同段拧到结论上
   - **括号用法**：反讽冷笑 ≤1、事务性注释不限、补充限定 ≤2，合计 ≤5 次/篇
   - **情绪一闪即收**：允许 1 句情绪峰值，下一句必须回到事实或推理
4. 通读检查密度和句式复杂度
5. 执行自检清单 → 进入"终稿补丁协议"

## 终稿补丁协议（写作/改写通用）

处理同任务内"草稿 → 终稿"阶段。目标：保留草稿的作者气质，只做补丁，不再写一遍。

1. **门槛**：必须基于当前任务草稿；没有草稿就先索取，不得凭空重写
2. **补丁预算（硬约束）**：只修 Top 5–10 个最致命命中；超限就回到生成/改写流程重做
3. **允许改动**：删模板路标句/空泛拔高/协作残留；修伪引述/不可追溯硬事实/标点错配；把信息塞回原语流，不改段落骨架
4. **禁止改动**：禁整段重写/全篇换词/结构重排；禁把锋利"我"视角和断言磨平为中性旁白；禁为"看起来更干净"删论证链和限定条件
5. **回滚闸门**：补丁后若读感更稳、更平、更像说明书，立即回滚到草稿，只保留必要补丁

## 翻译流程（结构保真）

1. 读 [translation-guardrails.md](references/translation-guardrails.md)（翻译任务最高优先级）
2. 锁定不可改动片段：代码块/内联代码/URL/email/路径/命令/参数/占位符/ID/版本号——原样保留
3. **Pass 1（快译）**：按原文顺序逐段译为简体中文，保留标题层级/段落边界/列表/引用/表格/缩进/空行，不合并/拆分/改序
4. **Pass 2（精修）**：不改结构、不增删、不改语气强度，把译文从"能懂"修到"像中文"——去翻译腔；口语转写则去噪；技术段落做符号化和单位标准化；按中西文混排补空格（详见 guardrails 6.x 节）
5. **专名策略**：稳定中文译名用译名；人名默认保留原文拼写；不确定就保留原文
6. **输出**：只输出译文正文；不输出解释/总结/对照表/译者注；不注入原文没有的立场和第一人称

## 审阅流程

1. 读 [style-dna.md](references/style-dna.md)
2. 按自检清单逐条对照标违反项
3. 按 [ai-trace-detector.md](references/ai-trace-detector.md) 做 AI 痕迹扫描，只列 Top 5–10 命中与修法（默认不做 24 项全量报告）
4. 给具体修改建议（附正确写法示例）

## 自检清单（核心 12 条；详细 AI 痕迹见 [ai-trace-detector.md](references/ai-trace-detector.md)）

写完/改完后逐条对照。定位风险并做最小修补，不得驱动整篇重写。

- [ ] **语法**：主谓搭配？关联词完整？文言虚词语义方向？
- [ ] **段落节奏**：1500-3000 字 9-13 段（科普可 10-14）？厚段 3-5？无连续 3 段 <90 字、连续 2 段一句成段、连续 3 段厚段？薄段都有功能？
- [ ] **句式**：每段多分句长句（30-80 字）为主？短句 ≤2/段？全文最短句 <40 字？1500-3000 字至少 3 处 <15 字短句？无连续 3 段以短句收尾？
- [ ] **语体**：消除所有网络口语/新媒体腔/标题党？读起来像书房写的？评论类每 200 字至少一次语体跳跃（书面↔口语↔文言）？
- [ ] **用词**：至少 5-8 个文言虚词自然嵌入？无空泛修饰词？无连续 3 个带形容词名词？无"A 而又 B"双形容词？
- [ ] **事实与引用预算**：每 500 字 ≥1 个事实锚点？跨学科锚点 ≤2 且至少 1 个深用？硬事实有来源否则降级为限定表述？
- [ ] **比喻密度**：每 1000 字 ≤2 个？无展开式类比/对价式比喻？无 AI 高频隐喻词（噪音/信号/底色/光谱/滤镜/解药/土壤/基因/拼图/镜像/路径/尺子/横截面/透镜/窗口/缩影）的比喻义用法？
- [ ] **情感**：情感通过判断句/否定句表达？激愤靠长句密度而非感叹号？感叹号 ≤2 个/篇且只在峰值句？
- [ ] **结构**：首尾呼应？无学术过渡词（此外/另外/总之/综上）？小标题数符合体裁限制？标题非对仗、非冒号？
- [ ] **冒号**：正文冒号 ≤2（1500-3000 字），只用于直接引语提示语？无"X 摆在那里 + 冒号"模板？
- [ ] **人称与括号**：「我」是知识分子式？「你」仅用于归谬反问 ≤2 次？反讽冷笑括号 ≤1？括号总量 ≤5？
- [ ] **AI 痕迹快扫**：按 [ai-trace-detector.md](references/ai-trace-detector.md) 修 Top 5–10 命中（伪口语化高频词/分析师讲解语姿/极值判断/关联句式堆叠/戏剧化揭露/伪学术腔/判断式陈述/二分对照壳/否定转折模板/宣传腔/模糊归因/协作口吻/知识截止免责）

## 命中即修红线（一行式黑名单）

以下模式在任何情况下都不得出现。每项详细机制和正误对照见 [ai-trace-detector.md](references/ai-trace-detector.md) 对应章节。

**结构类**
- 连续 3 段 <90 字 ‖ 连续 2 段一句成段 ‖ 连续 3 段厚段 ‖ "一观点一段"等长切片
- 标题：对仗式（X 与 Y）/ 冒号式（X：Y）/ 悬念党（震惊！/真相是…）
- 一个标题只管一个段落（即用 `##`/`###` 必须覆盖 ≥2 段正文）
- 单独一段写"挑战与机遇并存""未来可期""这只是开始"

**句式类**
- AI 典型句式：值得注意的是/不可否认/毋庸置疑/令人深思/引发关注
- 排比三段：不仅…而且…更…/既…又…还…/首先其次再次最后
- bullet point 和列表（散文化表达）
- 全长句无短句顿挫；或全短句无长河感
- 连续感叹号（每篇 ≤2 个，仅在情感峰值或化用引文）

**模板腔类**（详见 ai-trace-detector.md 第 7.1-7.6、9.1 节）
- 伪口语化高频词：拆一拆/拆解/梳理/剖析/解构/聊一聊/盘一盘/捋一捋/盘点一下/划重点/敲黑板/说白了/本质上/归根结底/简单来说/换个角度看/聚焦/洞察/深耕/赋能/助力/践行/驱动/构建/打造（偶尔 1 次可，2 次以上即改）
- 分析师讲解语姿/步骤承接：把 X 拆完/说完/看完/理完/弄清/厘清/讲完/聊完/分析完，再/便/就/回头/然后 Y
- AI 关联句式堆叠（全文 ≤2 次）：一旦…就 / 只有…才 / 无论…都 / 随着…的 / 正是因为…所以 / 正是…才 / 通过…来
- 戏剧化揭露修辞：扯下/撕下/揭下/掀开 + 遮羞布/面具/画皮/伪装/外衣；剥开/戳穿/戳破/揭穿 + 表象/真面目/本质/真相
- 极值判断（全文 ≤1 次）：最 X 的地方在于 / 真正 X 的是 / X 之处在于 / 更 X 的是（X = 残酷/可怕/讽刺/荒谬/吊诡）
- 伪学术腔：舆论场/话语场/权力场/公共场域 / 宏大叙事/官方叙事/主流叙事 / 摊开来看/摊开来说 / XX 初登台/粉墨登场 / 面子上是 X，里子是 Y / 底层逻辑/赛道/闭环/抓手/权柄（全文合计 ≤1）
- 判断式/定义式陈述堆叠（全文 ≤3）：A 是 B / A 不是 B / 所谓 A 即 B / A 本质上是 B + 绝对副词（必然/显然/毋庸置疑/根本/绝对/无疑/一定）
- 二分对照壳：不是 A 而是 B / 并非 A 而是 B / 不在于 A 而在于 B（清零；改为"A 固然…；但/而 B"不对称对照）
- 否定转折模板高频复用：先否定旧判断、再硬拐到新判断的结构体
- "X 摆在那里" / "先把 X 讲明" / "主要是两个目的，一是…二是…"
- 模糊归因：专家认为/研究表明/有人指出/普遍认为（无可追溯来源就降级为限定表述或删）

**语姿类**
- 协作交流痕迹：让我们/接下来/下面我将/希望这能帮助你
- 知识截止/模型身份免责：截至我的知识/作为 AI 模型/我无法访问最新信息
- 客服式谄媚：很高兴/荣幸/感谢你的提问/令人振奋的旅程
- 新闻体开局（时间+地点+人物+事件）
- 拟人化比喻开场（就像一个/更像是）
- 鸡汤收尾：在这个越来越…的世界里 / 或许…更有价值

**用词类**
- 网络腔：说白了/本质上就是/无非就是/半毛钱/翻车/收割/割韭菜/整活
- 宣传腔：深刻揭示了/充分体现了/具有重大意义
- 伪引述：把不可追溯的定性、口号、文件措辞塞进引号冒充原话
- 学术旁白句：对研究者而言/历史学家担心/学界认为（无新增事实就删）
