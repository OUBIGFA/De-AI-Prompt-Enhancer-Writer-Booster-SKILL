# de-AI-writing

## 0. 定位

本技能不单独定义新文风，默认继承 `../good-writing/SKILL.md`，再叠加“反模板硬约束”。

核心目标：删掉 AI 路标，不削信息厚度，不改写成讲义腔。

## 1. 最高优先级规则

### 1.1 默认改写模式 = 保真改写

除非用户明确要求“自由改写/允许增删重组”，否则默认执行保真改写：
- 不新增事实、案例、数据、结论
- 不删除原文核心论证链
- 尽量段落一一对应，只做语言与句式修补

### 1.2 输出形态

- 默认输出纯正文，不输出流程说明、检查清单、代码块
- 小标题默认允许但总数 `<= 3`
- 小标题禁止“背景/现状/挑战/展望”这类模板词

### 1.3 强制禁令（命中即失败）

以下禁令仅针对最终正文（逐字引文除外）：

- 禁用 `而是` 及其变体：`不是…而是`、`并非…而是`、`不在…而在`、`不只是…更是`、`并不是…而是`、`而是说`、`而是在`、`而是要`
- 禁用协作/教学路标：`接下来我们将`、`我们先来看`、`下面我们`
- 禁用客服/AI 身份口吻：`作为AI`、`截至我的知识`、`希望这能帮助你`
- 禁用解释/列举型冒号：最终正文中全角冒号 `：` 约 1500-3000 字目标 0-2（不强制清零，硬上限 `<= 2`），且仅允许直接引语提示语（如 `他说：“……”`）；其他场景命中即失败

## 2. 句式与语气约束

### 2.1 二人称约束（对齐 `.writer`）

- 默认不用 `你/你会`
- 如确需读者推演，只允许 1 次 `试想` 或 `当你` 句式
- 禁止连续两句出现第二人称

### 2.2 路标词预算

以下路标词全文合计建议 `<= 2`：`更关键`、`更要命`、`换句话说`、`事实上`、`值得注意`、`总之`、`与此同时`。

### 2.3 否定转折替换策略

当原句为“否定A -> 转折B”模板时，按以下顺序改写：
1. 直接写 B 的判决句，再补 1 句机制或事实
2. 用因果链：`之所以……是因为……`
3. 用层级句：`A 只能解释表层，B 才能解释结构后果`

禁止把模板简单替换为另一种模板路标。

## 3. 标点与结构控制

- 句号用于语义切割，分号用于并列推进，避免分号连发造成演讲腔
- 冒号按“极严预算”执行：约 1500-3000 字正文目标 0-2（不强制清零，硬上限 `<= 2`），并且仅限直接引语提示语使用
- 列举或解释关系优先改写为逗号并列、两句拆分或因果句，不写“标签：解释”式提纲句
- 优先厚段，不用“一个观点一段”的讲义切分

## 4. 工作流（执行顺序）

1. 按 `../good-writing/references/style-dna.md` 写或改出正文骨架
2. 按 `../good-writing/references/ai-trace-detector.md` 修 Top 5-10 命中
3. 执行“反模板快扫”：
   - `而是` 命中必须为 0
   - `你会` 命中必须为 0
   - 二人称总量可控（默认 0）
   - 路标词总量不超预算
   - 冒号 `：` 目标 0-2（不强制清零，硬上限 `<= 2`），且只出现在直接引语提示语
4. 复核保真改写要求：不增删核心信息

失败时优先局部修补，避免整篇重写。

## 5. 翻译模式（结构保真）

当任务为翻译时，遵循 `references/translation-guardrails.md`：
- 保留原文结构、层级、列表、表格、代码和链接
- 不增删信息，不改变原文立场强度
- 不注入额外第一/第二人称
- 输出只给译文正文

## 6. 审阅模式（显式触发）

仅当用户明确要求“审阅/检查/评分/列问题”时输出：
1. 风格贴合结论（是否接近 `.writer`）
2. Top 5-10 命中（片段 + 修法）
3. 关键风险（变平、变薄、变说明书）
