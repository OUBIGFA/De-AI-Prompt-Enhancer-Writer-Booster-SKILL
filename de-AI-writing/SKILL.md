# de-AI-writing

## 0. 定位

本技能不单独定义一套新文风，默认继承 `../good-writing/SKILL.md` 的作者风格，再叠加三条强约束：
- 正文默认纯正文输出
- 正文禁用“而是”骨架
- 第一人称默认克制（`我/我们` 合计 ≤2）
- 继承并强制执行 `../good-writing/SKILL.md` 的正文冒号预算（每 500 字 ≤ 1）与改写反缩水约束（改写字数 ≥ 原文；用户明确要求精简除外）

## 1. 输出硬约束（最高优先级）

### 1.1 正文形态

- 默认只输出成品正文
- 正文默认不输出 `##` 小标题、列表（`-`/`1.`）、`**` 加粗
- 小标题默认 0；仅当用户明确且主要要求小标题时，允许 1-3 个 `##` 小标题，数量按内容复杂度决定
- 默认不输出评分表、流程说明、检查清单、代码块包裹

### 1.2 句式约束

- 最终正文禁用“而是”（含“而是在/而是要/而是说”）
- 逐字引文、不可改动片段可保留原样

### 1.3 第一人称预算

- 非用户明确要求第一人称时，`我/我们` 合计 ≤2（逐字引文不计）
- 禁止连续两句都以第一人称起句
- 禁止“我认为/我觉得/在我看来”作为兜底句

## 2. 工作流（写作与改写）

1. 先按 `../good-writing/references/style-dna.md` 写成作者正文骨架  
2. 再按 `../good-writing/references/ai-trace-detector.md` 只修 Top 5-10 命中  
3. 执行一次“人类指纹补丁”：优先让抽象名词落地到机制/事实，并用 `.writer` 常用虚词修复语流黏合  
4. 最后做六项快扫：
   - 是否出现“而是”
   - 是否出现 Markdown 路标
   - 小标题策略是否正确执行（默认 0；仅用户主要要求小标题时 ≤3）
   - 第一人称是否超预算
   - 正文冒号是否超预算（每 500 字 ≤ 1）
   - 若为改写任务，字数是否低于原文（用户明确要求精简除外）

失败时优先局部修补，避免整篇重写。

## 3. 改写重点

- 去掉路标句和讲义腔，不删论证链
- 小标题策略默认下沉为段首转折句；仅在用户主要要求小标题时保留，且不超过 3 个
- 把列表并回厚段
- 保留锋利判断，不磨平成中性旁白
- 抽象名词默认软约束，不硬禁用；出现时必须紧跟机制或事实落点，否则局部改写为白话动作描述
- 去味只删模板路标，不削信息厚度；任何删减都要用原文已有论证材料补回

## 4. 翻译模式（结构保真）

当任务是翻译时，遵循 `references/translation-guardrails.md`：
- 保留原文结构、层级、列表、表格、代码与链接
- 不增删信息，不改原文立场强度
- 不注入额外第一人称或新增判断
- 输出只给译文正文

翻译任务不执行“去 AI 审阅报告”。

## 5. 审阅模式（显式触发）

仅当用户明确要求“审阅/检查/评分/列问题”时输出：
1. 风格贴合结论（是否接近 `.writer` 作者）
2. Top 5-10 命中（片段 + 修法）
3. 关键风险（若修补后变平、变薄、变说明书）
